In this exam you are going to develop a web application to help ITS Help Deskto keep track of service requests. Your application  must use the MVC architecture as discussed in class, and the application must meet the following requirements:

  • Servlets cannot be used to generate HTML content.
  • No scripting elements are allowed in JSP pages.

1. Display Service Requests

The application should have a Requests page that lists all the service requests that are currently managed by the application. For example:

Request Time Requested By Service Type Status  
2013-02-14 8:15:33AM John Phone Open Details Update
2013-02-18 2:11:12PM Jane Computer Assigned Details Update
2013-01-12 3:15:16PM Jack Account Completed Details

Create Service Request

We assume that there are four service types: Computer Service, Network Service, Phone Service, and Account Service. The status of a service request can be Open, Assigned, or Completed. When a service request is created, its status is Open. An ITS Help Desk personnel can assign the service request to a technician and set the status of the request to Assigned. After the technician completes the service, he or she can update the request to set its status to Completed.

Note that the request listing should show all the Open requests first, followed by the Assigned requests, and then the Completed requests. Requests with the same status should be ordered by request time in ascending order, and Request Time should be formatted as shown in the example above.

Clicking on the Create Service Request link should take a user to the Create Request page where a new service request can be created. After the user fills out the form and clicks the Create button, the application should create a service request and redirect the user back to the Requests page.

3. Display Service Request Details

Clicking on the Details link of a request should take the user to the Request Details page where all the information about the service request is shown. For example:

Computer Service requested by Jane at 2013-02-18 2:11:12PM.

Status: Assigned to Bob

Description of Work Requested:

There seems to be a virus on my computer.

All Requests

Note that if the status of the request is Assigned or Completed, it should show the name of the technician who was assigned to or completed the request. Clicking on the All Requests link should take the user back to the Requests page.

4. Update Service Request Status

A user, presumably an ITS supervisor or technician, can click on the Update link of a service request to update the status of the request. If the status of a request is Open, an ITS supervisor can assign the request to one of the three ITS technician: Amy, Bob, and Tom. For example:

Phone Service requested by John at 2013-02-14 8:15:33AM.

Status: Open

Description of Work Requested:

My office was moved from E&T A101 to E&T B202. Please transfer my phone number (ext 1234) to my new office.

Assign this request to

If the status of a request is Assigned, the ITS technician who completed the service request can change the status to Completed. For example:

Computer Service requested by Jane at 2013-02-18 2:11:12PM.

Status: Assigned to Bob

Description of Work Requested:

There seems to be a virus on my computer.

Change the status to Completed? Yes No

Clicking on Yes should change the status to Completed and redirect the user to the Requests page; clicking on No simply takes the user to the Requests page.

If the status of a request is Completed, the Update link for that request on the Requests page should be removed.
