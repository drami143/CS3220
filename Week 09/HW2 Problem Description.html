CS3220 Homework 2

Please deploy your application on CS3, and package your project in a zip file and submit it to Canvas. The open-source software tool 7-Zip can be used to create the zip file. Please also create and submit an HTML file hw2.html which contains a hyperlink to your application on the CS3 server.

In this assignment you will continue to work on the art school application from Lab 9 to add functions to manage students and groups. Your HW2 code will conflict with your Lab 9 code on CS3, so do NOT deploy your HW2 on CS3 until a) Lab 9 is graded and you do not plan to contest the grade, and b) you have removed Lab 9 code from CS3.

1. List Student (15pt)

Modify the student listing page so it also shows the group a student belongs to and an Edit link (or button). For example:

New Student

Student Age Parent Email Group  
John 4 Sue sue@gmail.com Minnows Edit
Jane 9 Sue sue@gmail.com Dolphines Edit
Luke 5 Steve steve@gmail.com Minnows Edit
Tina 7 Paula paula@gmail.com   Edit

If a student does not belong to any group, the group column for the student is empty.

2. Edit Student (40pt)

Clicking on an Edit link (or button) on the student listing page should take the user to a page where the information of the corresponding student can be edited. For example:

Name
Birth Year
Parent Name
Parent Email
Group

Note that the form must be pre-filled with the current information of the student. In particular, if the student belongs to a group, the group should be selected in the group dropdown list. After the information of the student is updated, the user should be redirected to the student listing page.

3. Edit Group (30pt)

Similar to the student listing page, modify the group listing page to add an Edit link (or button) to each group. When an Edit link (or button) is clicked, the user should be taken to a page where the group information can be edited. For example:

Name
  • John [Remove]
  • Luke [Remove]

Note that the Edit Group page shows a form to change the name of the group, as well as links (or buttons) that can be used to remove individual students from the group. After the name of the group is updated, the user should be redirected to the group listing page.

4. Remove Student from Group (15pt)

Clicking on a Remove link (or button) on the Edit Group page will remove the corresponding student from group. After the student is removed from the group, the user should be redirected back to the Edit Group page.

5. (40pt) Use Bootstrap to style the application. In particular,

  • Tables must use Bootstrap table styles.
  • Forms must use Bootstrap form styles.
  • Links and buttons must use Bootstap Buttons. Note that you can make a link to look like a button by giving it the Bootstrap btn class.
  • Every page except the home page should use Bootstrap Breadcrumb so the user can easily navigate back to previous pages. For example, the Edit Student page should have breadcrumb to go to the home page and the student listing page.

You may choose to use either Boostrap 5.x or 4.x (and when you look up Bootstrap documentation on their website, be aware of which version the documetnation is for.)
